已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,4,4,5,6,7] 在变化后可能得到：
若旋转 4 次，则可以得到 [4,5,6,7,0,1,4]
若旋转 7 次，则可以得到 [0,1,4,4,5,6,7]
注意，数组 [a[0], a[1], a[2], ..., a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]] 。
给一个可能存在 重复 元素值的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 

***旋转数组***


官方答案：
①如果pivot的值小于high，那pivot在最小值右边，所以区间右半部分可以去掉，但是由于只知道pivot的比high的小，不知道pivot的到底是不是最小值，所以不能丢掉，只能high=pivot。
②如果pivot的大于high，说明pivot在最小值的左边，在最小值左边的部分全部都大于右半部分，这时候可以肯定pivot绝对不是最小值，所以可以舍弃这个pivot的值，low=pivot+1。
③如果pivot的值等于high的值，它可能在最小值右边并且右边有好几个相等的值，这时候不能就认为得到结果了，但是可以确定high的值不是唯一的值，即使扔掉了也有pivot的值一样，就high--；它可能在最小值左边并且像3 3 3 1 3 3 3 3 3这样，这时候我不知道high是不是最小值，但因为pivot的值一样所以丢了也无所谓，也能high--。
④因为一个是high=pivot，一个是low=pivot+1，所以在找到答案后最终只可能low==high而不会>，所以条件是< (如果是low>high条件就是<=)
⑤直到最后停止循环可以随便找个例子执行到最后看看high和low指到哪里，拿指着最小值的返回nums[?]。

只有两个地方要判断：
第一个：两个边界是=mid还是=mid±1（具体题目分析，不能丢掉目标值）
第二个：<还是<=（看找到答案时左右边界的关系，是left==right还是left>right）

```cpp
class Solution
{
public:
    int findMin(vector<int> &nums)
    {
        int low = 0;
        int high = nums.size() - 1;
        while (low < high)
        {
            int pivot = low + (high - low) / 2;
            if (nums[pivot] < nums[high])
            {
                high = pivot;
            }
            else if (nums[pivot] > nums[high])
            {
                low = pivot+1;
            }
            else
            {
                high--;
            }
        }
        return nums[low];
    }
};
```
