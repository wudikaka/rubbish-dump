已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：  
若旋转 4 次，则可以得到 [4,5,6,7,0,1,2]  
若旋转 7 次，则可以得到 [0,1,2,4,5,6,7]  

注意，数组 [a[0], a[1], a[2], ..., a[n-1]] 旋转一次的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]] 。  

给你一个元素值 互不相同 的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 。  

***旋转数组***


不停二分，每次二分的nums[mid]有两种情况：
①nums[mid] > nums[right] -> 类似4,5,6,7,0,1,2的mid取到7，right取到2 -> min在右边  
②nums[mid] < nums[right] -> 类似5,6,7,0,1,2,4的mid取到0，right取到4 -> min在左边或就是mid  
                             或0,1,2,4,5,6,7完全递增  
每次二分完就往min可能在的地方缩小，每次缩小的时候left始终在min左边，直到最后left==mid==right时已经在min下标了，到最后停止二分后发现left正好指向min

```cpp
class Solution
{
public:
    int findMin(vector<int> &nums)
    {
        int l = 0;
        int r = nums.size() - 1;
        while (l <= r)
        {
            int mid = l + (r - l) / 2;
            if (nums[mid] > nums[nums.size() - 1]) // 最小值在右边
            {
                l = mid + 1;
            }
            else // 最小值在左边
            {
                r = mid - 1;
            }
        }
        return nums[l];
    }
};

官方方法：
思路同上，但实现方式不一样
如果是一般情况，最右边那个不可能是最小值，不用搜了，如果是特殊情况，就是只有最右边的是最小值，左边全部增且大于它，这情况最后l也会落在最右边，所以虽然是左闭右开但high依旧从n-1开始  
因为区间是[low,high)，high是开区间，high这个位置本来就不搜，每次high=pivot缩减到这里已经相当于忽略high上的了，所以不用high=pivot-1


```cpp
class Solution 
{
public:
    int findMin(vector<int>& nums) 
    {
        int low = 0;
        int high = nums.size() - 1;
        while (low < high) 
        {
            int pivot = low + (high - low) / 2;
            if (nums[pivot] < nums[high]) 
            {
                high = pivot;
            }
            else 
            {
                low = pivot + 1;
            }
        }
        return nums[low];
    }
};
