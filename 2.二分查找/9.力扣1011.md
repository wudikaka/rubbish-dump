传送带上的包裹必须在 days 天内从一个港口运送到另一个港口。
传送带上的第 i 个包裹的重量为 weights[i]。每一天，我们都会按给出重量（weights）的顺序往传送带上装载包裹。我们装载的重量不会超过船的最大运载重量。
返回能在 days 天内将传送带上的所有包裹送达的船的最低运载能力。

***二分答案***
*最大值最小\最小值最大*

官方方法：
因为求的是船单天可以运输重量的最小值，所以我的left/right/mid应该表示重量，发现船单天运输能力不能低于最重的包裹，所以确定左边界；发现单天最大运输不可能超过所有包裹重量和，不然浪费运力，所以确定右边界。总之，二分查找的数组其实就是一个[max,sum]的数组。  

求出mid，从前往后让元素依次累加，超过mid就下一轮，看看累加完要几轮，就是几天。如果<=目标天数，就是运力过大或正好，那就向左缩减，但可能mid就是答案，所以不能丢，right=mid，但是因为是<=，到最后必定left=right+1，所以即使是right=mid-1丢掉了这个答案，但最后left也可能落在这上面；如果>，就是运力太小，左边不可能有答案，直接右移。最终根据下标的含义，right右边全是need<=days的，并且递增，找个最小的即可，可能不会刚刚好，但应该是能=days。

单天包裹重量累加逻辑：取每天的元素，判断当天的和以前累加的有没有超过规定的单天运量(mid)，没超过就累加，超过了就清空并且增加一次所需天数，本轮没放进去的可以在本轮清空后继续存入，这里设计很巧妙，因为sum的位置在清空之后，所以可以很巧地接住上一天没放进去的包裹重量  
等价写法：  
```cpp
int need = 1, cur = 0;
for (int weight: weights) {
    if (cur + weight > mid) {
        need++;          // 新开一天
        cur = weight;    // 新的一天直接装这件货
    } else {
        cur += weight;   // 今天还能装
    }
}
```

答案代码：  
```cpp
class Solution
{
public:
    int shipWithinDays(vector<int> &weights, int days)
    {
        // 确定二分查找左右边界
        int left = *max_element(weights.begin(), weights.end()), 
        right = accumulate(weights.begin(), weights.end(),0);
        while (left <= right)
        {
            int mid = (left + right) / 2;
            // need 为需要运送的天数
            // cur 为当前这一天已经运送的包裹重量之和
            int need = 1, cur = 0;
            for (int weight : weights)
            {
                if (cur + weight > mid)
                {
                    ++need;
                    cur = 0;
                }
                cur += weight;
            }
            if (need <= days)
            {
                right = mid-1;
            }
            else
            {
                left = mid + 1;
            }
        }
        return left;
    }
};
```
